# 根目录文件规范检查

检查项目根目录是否符合以下规范：

## 规范要求

1. **根目录只允许以下 Markdown 文件（不区分大小写）**：
   - README.md
   - CLAUDE.md

2. **测试脚本文件规范**：
   - 所有 `test_*.py`、`check_*.py` 等测试/检查脚本必须放在 `scripts/` 目录
   - 根目录不应出现这类脚本文件

3. **其他 .md 文件**：
   - 除上述两个文件外，其他 .md 文件不允许出现在根目录
   - 应放在 `docs/` 或其他子目录

## 检查指令

请执行以下检查：

1. 列出根目录下所有 `.md` 文件，检查是否只有 README.md 和 CLAUDE.md
2. 列出根目录下所有符合 `test_*.py`、`check_*.py` 模式的 Python 文件
3. 如果发现违规文件，输出警告信息，格式如下：

```
⚠️  根目录文件规范检查失败:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
❌ 测试脚本不应在根目录: xxx.py (应移至 scripts/)
❌ 根目录只允许 README.md 和 CLAUDE.md: xxx.md
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📝 建议操作:
   - 测试脚本请移至: scripts/
   - 其他文档请移至: docs/
   - 只保留 README.md 和 CLAUDE.md 在根目录
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

4. 如果一切正常，不需要输出任何内容

## 注意事项

- 使用 Glob 工具检查文件，不要使用 Bash 命令
- 文件名比较应不区分大小写
- 这只是检查和提醒，不要自动移动或删除文件
